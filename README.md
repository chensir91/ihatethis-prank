# 我好恨 - 恶作剧应用

## 功能说明

### 基础功能
- 图标透明，名称透明（桌面上几乎看不见）
- 开机自启动
- 仅在指定时间段触发显示
- 屏幕正中央显示"我好恨"三个大字
- 从触发开始计时，颜色逐渐加深
- 达到设定时间后达到最深的血红色
- 超过结束时间后自动消失

### 新增功能（v2.0）
- ✅ **可自定义触发时间**：自由设置开始时间和结束时间
- ✅ **可自定义渐变时长**：1-600分钟可调
- ✅ **支持跨天设置**：例如晚上22:00到次日早上6:00
- ✅ **实时模拟预览**：拖动滑块查看任意进度的显示效果
- ✅ **动画播放预览**：一键播放完整渐变动画

### 新增功能（v3.0）
- ✅ **打开直接进入设置**：启动APP直接进入设置界面，更方便操作
- ✅ **全局悬浮窗显示**：血色文字悬浮在所有应用上方，不影响正常使用
- ✅ **触摸穿透**：悬浮窗不拦截触摸事件，可以正常操作下方的应用
- ✅ **后台服务运行**：开机自启动后台服务，定时检查触发时间
- ✅ **悬浮窗预览**：可直接预览悬浮效果，3秒后自动关闭
- ✅ **权限检测**：自动检测悬浮窗权限，引导用户开启

## 使用说明

### 进入设置界面
打开APP直接进入设置界面，可以配置所有参数。

### 设置项说明
| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 触发开始时间 | 每天开始显示的时间点 | 00:00 |
| 触发结束时间 | 每天结束显示的时间点 | 04:44 |
| 渐变时长 | 从透明到完全血色所需时间（分钟） | 240 |

### 模拟预览功能
- **拖动进度条**：实时查看任意百分比的颜色深度
- **播放动画**：2.5秒快速播放完整渐变过程
- **停止**：暂停动画播放

### 悬浮窗控制
1. **开启悬浮窗权限**：首次使用需要点击权限状态提示，跳转到系统设置开启悬浮窗权限
2. **启动服务**：点击"启动服务"按钮，后台服务开始运行，到设定时间自动显示悬浮文字
3. **停止服务**：点击"停止服务"按钮，停止后台服务和悬浮显示
4. **预览悬浮效果**：点击"预览悬浮效果"按钮，可以预览3秒的悬浮效果

### 悬浮窗特点
- **全局显示**：悬浮在所有应用上方，包括桌面、游戏、视频等
- **触摸穿透**：不会拦截触摸操作，可以正常使用下方的应用
- **渐变效果**：从透明逐渐变为血红色，营造恐怖氛围
- **自动隐藏**：超过结束时间后自动消失

## 编译方法

### 方法一：使用 GitHub Actions 自动构建（推荐）
1. 在 GitHub 上创建新仓库
2. 将本项目代码推送到 GitHub 仓库
3. 推送后 GitHub Actions 会自动开始构建
4. 在仓库的 Actions 页面查看构建进度
5. 构建完成后，在 Artifacts 中下载 `app-debug-apk`

### 方法二：使用 Android Studio 编译
1. 下载并安装 Android Studio
2. 打开 Android Studio，选择 "Open an existing project"
3. 选择本项目文件夹（Ihatethis）
4. 等待 Gradle 同步完成
5. 点击菜单 Build -> Build Bundle(s) / APK(s) -> Build APK(s)
6. 编译完成后，APK 文件位于 `app/build/outputs/apk/debug/app-debug.apk`

## 推送到 GitHub 的步骤

```bash
# 1. 在 GitHub 上创建新仓库（不要初始化 README、.gitignore 等）
# 2. 在本地项目目录执行：
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

## 安装说明

1. 将 APK 文件传到手机
2. 在手机上点击安装
3. 安装后打开应用，进入设置界面
4. **开启悬浮窗权限**：点击权限状态提示，跳转到系统设置开启悬浮窗权限
5. 点击"启动服务"按钮启动后台服务
6. 开启"开机自启动"权限（在手机管家/设置中找到应用，开启自启动权限）
7. 设置好触发时间和渐变时长后即可正常使用

## 注意事项

- 本应用仅供娱乐/恶作剧使用
- 首次安装后必须手动开启悬浮窗权限才能正常显示
- 不同品牌手机的自启动权限和悬浮窗权限设置位置不同，请自行查找
- Android 10+ 可能需要额外的后台活动权限
- 前台服务通知：为了保证服务稳定运行，会在通知栏显示一个低优先级通知
- 设置修改后需要重启服务才能生效
